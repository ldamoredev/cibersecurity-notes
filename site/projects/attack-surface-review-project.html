Attack Surface Review Project

Scope

Take one real deployed app and map domains, subdomains, ports, admin interfaces, deprecated endpoints, public storage, reverse proxies, and CI/CD exposures.

Main Branches Involved

  • Attack Surface Mapping
  • Offensive Security / Recon
  • Networking
  • API Security

Architecture / System Shape

  • Root domains and DNS providers
  • Public ingress points
  • Reverse proxies and load balancers
  • Application endpoints and API versions
  • Public storage buckets or object stores

Attack Surface

  • External Attack Surface
  • Service Enumeration
  • Admin Interface Discovery
  • Endpoint Discovery
  • Exposed Storage

Testing Hypotheses

  • Expected architecture differs from reachable assets.
  • Some stale DNS or subdomains still point to third-party services.
  • Admin interfaces are reachable from the public internet.
  • Old API versions or docs remain exposed.

Fixes / Hardening Decisions

  • Remove stale records
  • Restrict admin surfaces
  • Close deprecated endpoints
  • Harden ingress and storage permissions
