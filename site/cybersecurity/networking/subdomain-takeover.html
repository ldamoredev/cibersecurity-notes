Subdomain Takeover

Definition

Subdomain takeover happens when a DNS record still points to an external service that is no longer owned or provisioned by the intended organization, allowing an attacker to claim that target and serve content under the abandoned subdomain.

Why It Matters

It turns stale DNS into real attack surface. A forgotten record can become a trusted-looking host for phishing, session theft, script hosting, cookie scope abuse, or internal trust bypass assumptions.

Attacker Perspective

Attackers search for dangling CNAMEs, deprovisioned SaaS targets, expired cloud app mappings, and provider-specific error pages that indicate a hostname can still be claimed.

Defender Perspective

Defenders should treat DNS ownership as part of asset lifecycle management. Every public hostname needs an owner, a live target, and a removal path when the backing service is retired.

Practical Examples

  • blog.example.com still points to a deleted GitHub Pages or Azure resource that an attacker can re-register.
  • A staging hostname remains mapped to a decommissioned SaaS app and becomes attacker-controlled.
  • A takeover-ready hostname is later referenced by employees, documentation, or automation because the DNS name still looks legitimate.
