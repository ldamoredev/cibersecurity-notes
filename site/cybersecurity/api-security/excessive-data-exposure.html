Excessive Data Exposure

Definition

Excessive data exposure happens when an API returns more data than the client actually needs, often trusting the frontend to hide or ignore sensitive fields.

Why it matters

APIs are machine-readable, so extra fields are easy to inspect and diff. Even if the UI never renders them, they may still expose internal identifiers, secrets, personal data, or business-sensitive state. This is primarily a read-side problem: the server returns too much. mass-assignment is the corresponding write-side risk where the server accepts too much.

Attacker perspective

Attackers compare: - what the UI displays - what the raw response contains - what different roles receive - what changes across endpoints and versions

Defender perspective

Defenders should: - return only fields the client needs - define DTOs intentionally instead of serializing internal models - review role-based field visibility - test raw responses, not just UI output

Practical examples

  • the response includes password hashes, internal IDs, or IP addresses
  • an admin-only field appears for normal users but is simply hidden in the UI
  • old API versions still return verbose internal objects
