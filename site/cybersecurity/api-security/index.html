API Security Index

Purpose

This index is the root entry point for the API-security branch of the cybersecurity vault.

Use it to: - navigate the API-security notes - understand the order of study - connect web-security concepts to API-specific risk models - strengthen backend security intuition with object-, function-, and property-level authorization thinking

Use Reference Registry — API Security as the source of truth for references in this branch. Return to Cybersecurity Index for root navigation across branches.

Branch maintenance notes

  • Atomic notes in this branch should follow atomic-note.
  • Prefer count-based framing hooks in How it works or Variants and bypasses.
  • Use reference-registry-api-security before adding or changing references.
  • Preserve the split between object authorization (broken-object-level-authorization), property authorization (broken-object-property-level-authorization), function authorization (broken-function-level-authorization), and broad policy framing (authorization).
  • Practical labs should use owned APIs, local labs, or intentionally vulnerable training targets.
